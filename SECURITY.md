# Security policy

KoT currently contains only documents and drafts (no deployed code). This policy establishes a responsible disclosure path as the project evolves.

## Reporting a vulnerability

If you believe you have found a security vulnerability related to this repository or any future reference implementation:

* Do not open a public issue with exploit details.
* Contact the maintainers (security contact is not yet set).

Until a dedicated security email is published, open a GitHub issue titled **"Security report (private channel requested)"** with minimal details and request a private channel.

## Supported versions

No versions are currently supported (no software releases).
